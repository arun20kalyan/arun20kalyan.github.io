Text Classification using Naive Bayes Classifier

 

 

A classifier refers to a type of machine learning algorithm designed to differentiate between different entities by analysing specific characteristics or attributes.

 

The Naive Bayes classifier is a machine learning technique that involves probabilistic analysis to perform classification tasks. At its core, the classifier relies on the principles of Bayes theorem and conditional independence to make predictions and categorize data.

 

Bayes Theorem:

 

 

 

Bayes theorem can be utilized to calculate the likelihood of the occurrence of hypothesis when evidence has already taken place. The key assumption in this approach is that the predictors or features are not correlated with each other, meaning the presence of one feature does not impact the other. This assumption is what makes it "naive".

 

The process of assigning a label or category to a given text is referred to as text classification. This task is commonly used in various applications, such as sentiment analysis, natural language inference, and evaluating grammatical accuracy.

 

 

Conditional Probability:

It is a measure of the likelihood of one event occurring given that another event has already occurred. It is typically expressed in relation to two related random variables, such as X and Y, where events A and B correspond to specific outcomes of those variables.

 

 

 

Conditional Independence:

Conditional independence in probability theory defines circumstances in which an observation is unnecessary or redundant when assessing the plausibility of a hypothesis.

import pandas as pd

from sklearn.model_selection import train_test_split

 

# Load the dataset into a pandas dataframe

df = pd.read_csv('rt_reviews.csv' , encoding='iso-8859-1')

 

# Split the dataset into train and test sets

train, test = train_test_split(df, test_size=0.3, random_state=42)

 

# Split the test set into test and development sets

test, development = train_test_split(test, test_size=0.5, random_state=42)

 

# Save the train, development, and test sets to separate CSV files

train.to_csv('train.csv', index=False)

development.to_csv('development.csv', index=False)

test.to_csv('test.csv', index=False)

 

 

Loading the dataset from a CSV file, and then splits it into three sets - train, test, and development - using the train_test_split function from the sklearn.model_selection module. The train set contains 70% of the original dataset, while the test and development sets each contain 15%. The splits are reproducible due to the random_state parameter being set to 42.

The resulting CSV files can be utilized for machine learning model training, validation, and testing. The train set is used for model training, the development set is used for fine-tuning the model's hyperparameters and evaluating performance, and the test set is used for final evaluation of the model's performance.

Lastly, the encoding parameter is set to 'iso-8859-1' to make sure the text data in the CSV file is correctly decoded. Depending on the original file's encoding, a different encoding may be necessary.

Here we are generating a list of all unique words from the "Review" column in the "train" dataset, which is referred to as the "vocabulary". To accomplish this, the code first imports the "string" module to access a list of all punctuation characters. It then creates an empty list called "vocab" to store the vocabulary.

Then we loop through each row in the "Review" column of the "train" dataset. For each row, it converts all text to lowercase using the "lower()" function, removes all punctuation using the "translate()" method with the "maketrans()" function from the "string" module, and then splits the resulting text into individual words using the "split()" method. The individual words are stored in a list called "words".

Finally we add the individual words from the "words" list to the "vocab" list using the "extend()" method. This creates a complete list of unique words from the "Review" column of the "train" dataset, with all punctuation removed and all text converted to lowercase.

 

 

from collections import Counter

 

# Create empty dictionaries to store word counts and freshness counts

word_counts = {}

freshness_counts = {'fresh': 0, 'rotten': 0}

 

# Iterate over each row in the train dataframe

for index, row in train.iterrows():

    # Get the Freshness and Review values for the row

    freshness = row['Freshness']

    review = row['Review']

    # Convert the text to lowercase

    review = review.lower()

    # Remove punctuation

    review = review.translate(str.maketrans('', '', string.punctuation))

    # Split the text into individual words

    words = review.split()

    # Update the word and freshness counts

    for word in words:

        if word in word_counts:

            word_counts[word][freshness] += 1

        else:

            word_counts[word] = {'fresh': 0, 'rotten': 0}

            word_counts[word][freshness] += 1

    freshness_counts[freshness] += 1

 

# Calculate the total number of reviews

total_reviews = freshness_counts['fresh'] + freshness_counts['rotten']

 

# Calculate the conditional probability of each word

word_probs = {}

for word in word_counts:

    fresh_prob = word_counts[word]['fresh'] / freshness_counts['fresh']

    rotten_prob = word_counts[word]['rotten'] / freshness_counts['rotten']

    word_probs[word] = {'fresh': fresh_prob, 'rotten': rotten_prob}

 

# Calculate the prior probabilities of Freshness

fresh_prior_prob = freshness_counts['fresh'] / total_reviews

rotten_prior_prob = freshness_counts['rotten'] / total_reviews

 

 

 Here we are build a Naive Bayes model for classifying the "Freshness" value of movie reviews based on the words used in the reviews. First, we calculate the probabilities of each word in the "Review" column of the "train" dataset, given its "Freshness" value. We do this by importing the "Counter" module from the "collections" library, and initializing an empty dictionary called "word_counts" to store the counts of each word given its "Freshness" value, and a dictionary called "freshness_counts" to store the counts of each "Freshness" value in the dataset.

Then, we iterate over each row in the "train" dataset. For each row, we gets the "Freshness" and "Review" values, converts the text to lowercase, removes all punctuation, and splits the resulting text into individual words. Then we update the "word_counts" dictionary with the count of each word given its "Freshness" value, and update the "freshness_counts" dictionary with the count of each "Freshness" value.

Finally, we calculate the conditional probability of each word given its "Freshness" value, and store it in a dictionary called "word_probs". We also calculates the prior probabilities of "Freshness" and stores them in variables called "fresh_prior_prob" and "rotten_prior_prob" to use these values to initialise our each freshness values for dev dataset.

 

 

 

# Create a list to store the predicted Freshness values

predictions = []

 

# Iterate over each row in the dev dataframe

for index, row in development.iterrows():

    # Get the Review text for the row

    review = row['Review']

    # Convert the text to lowercase

    review = review.lower()

    # Remove punctuation

    review = review.translate(str.maketrans('', '', string.punctuation))

    # Split the text into individual words

    words = review.split()

    # Initialize the probabilities for each Freshness value

    fresh_prob = fresh_prior_prob

    rotten_prob = rotten_prior_prob

    # Update the probabilities based on the words in the review

    for word in words:

        if word in word_probs:

            fresh_prob *= word_probs[word]['fresh']

            rotten_prob *= word_probs[word]['rotten']

    # Determine the predicted Freshness value

    if fresh_prob > rotten_prob:

        predictions.append('fresh')

    else:

        predictions.append('rotten')

 

# Calculate the accuracy of the predictions

correct_predictions = (predictions == development['Freshness']).sum()

total_predictions = len(predictions)

accuracy = correct_predictions / total_predictions

print(f'Dev accuracy: {accuracy:.2%}')

 

 

 

Now we are predicting the Freshness values of reviews in the "development" dataset using a Naive Bayes classifier trained on the "train" dataset. We do this by creating an empty list called "predictions" to store the predicted Freshness values for each row in the "development" dataset. Then, we apply the same pre-processing steps used during training to prepare the review text for classification.

Next, the probabilities for each Freshness value are initialized to their prior probabilities calculated in the training phase. The probabilities for each word in the review are then updated by multiplying them with the conditional probabilities of Freshness values for the corresponding words.

Finally, the predicted Freshness value for the review is determined by comparing the fresh and rotten probabilities and choosing the one with higher probability. The predicted Freshness value is appended to the "predictions" list.

Once all the reviews in the "development" dataset have been classified, we calculate the accuracy of the predictions by comparing them with the actual Freshness values in the "development" dataset.

# Set the value of k for Laplace smoothing

k = 1

 

# Calculate the conditional probability of each word with Laplace smoothing

word_probs = {}

for word in word_counts:

    fresh_count = word_counts[word]['fresh'] + k

    fresh_prob = fresh_count / (freshness_counts['fresh'] + k * len(word_counts))

    rotten_count = word_counts[word]['rotten'] + k

    rotten_prob = rotten_count / (freshness_counts['rotten'] + k * len(word_counts))

    word_probs[word] = {'fresh': fresh_prob, 'rotten': rotten_prob}

 

 

 

Here we are modifying the calculation of the conditional probabilities of each word in the "train" dataset by adding Laplace smoothing.

The value of k is set to 1 for Laplace smoothing. Then we loop through each word in the "word_counts" dictionary and computes the conditional probability of the word with Laplace smoothing for both Freshness values.

The fresh_count is the count of the word in reviews with Freshness value "fresh" plus k, and fresh_prob is the resulting probability divided by the sum of all Freshness values plus k times the number of unique words in the dataset. The same calculation is performed for rotten_count and rotten_prob.

The resulting probabilities are stored in the "word_probs" dictionary for each word and Freshness value.

This will help us to handle the problem of zero probability in Naïve Bayes.

Log-odds ratio:

The log-odds ratio is a statistical measure that compares the odds of an event in one group (e.g., the probability of a word appearing in positive reviews) to the odds of the same event in another group (e.g., the probability of the same word appearing in negative reviews). It is calculated by taking the logarithm of the ratio of the probabilities of the event occurring in the two groups. In the context of sentiment analysis, the log-odds ratio is used to determine which words are most strongly associated with positive or negative reviews. A high log-odds ratio for a particular word suggests that the word is more likely to appear in reviews with a certain sentiment, while a low log-odds ratio indicates that the word is less informative for predicting sentiment. By sorting words by their log-odds ratio and selecting the top words for each sentiment, we can identify the most predictive words for positive and negative reviews.

 

 

import numpy as np

# Create empty dictionaries to store log-odds ratio for each word

log_odds_ratio = {'fresh': {}, 'rotten': {}}

 

# Calculate the log-odds ratio for each word

for word in word_probs:

    fresh_prob = word_probs[word]['fresh']

    rotten_prob = word_probs[word]['rotten']

    log_odds_ratio['fresh'][word] = np.log(fresh_prob / rotten_prob)

    log_odds_ratio['rotten'][word] = np.log(rotten_prob / fresh_prob)

 

# Sort the words by their log-odds ratio and take the top 10 for each class

top_words = {'fresh': [], 'rotten': []}

for freshness in ['fresh', 'rotten']:

    top_words[freshness] = sorted(log_odds_ratio[freshness], key=log_odds_ratio[freshness].get, reverse=True)[:10]

   

# Print the top 10 words for each class

print("Top 10 words that predict 'Fresh' reviews:")

print(top_words['fresh'])

print("\nTop 10 words that predict 'Rotten' reviews:")

print(top_words['rotten'])

 

 

Now we are calculating the log-odds ratio for each word in the "word_probs" dictionary and stores them in the "log_odds_ratio" dictionary for both Freshness values.

We do this by creating an empty "log_odds_ratio" dictionary with keys for both Freshness values. Then, we iterate it over each word in the "word_probs" dictionary and calculates the log-odds ratio by taking the natural logarithm of the ratio of the Freshness probabilities for the word.

The resulting log-odds ratios for each word and Freshness value are stored in the "log_odds_ratio" dictionary.

Next, we sort the words in the "log_odds_ratio" dictionary for each Freshness value in descending order based on their log-odds ratio and selects the top 10 words for each Freshness value. It stores these top words in the "top_words" dictionary with keys for both Freshness values.
